# แผนการจัดการเรียนรู้บูรณาการแบบ Active Learning (GPAS 5 Steps)
**รายวิชาเทคโนโลยี (วิทยาการคำนวณ) กลุ่มสาระการเรียนรู้วิทยาศาสตร์และเทคโนโลยี ชั้นประถมศึกษาปีที่ 6**

**ชื่อหน่วยการเรียนรู้:** วิศวกรตัวน้อย: สืบค้นภูมิปัญญาท้องถิ่น ออกแบบและสร้างบ้านด้วย Minecraft
**เวลาจัดกิจกรรม:** 5 ชั่วโมงเรียน + ภาระงานนอกเวลา (สัมภาษณ์ปราชญ์ชาวบ้าน)

**ชื่อผู้สอน:** นายภาณธร ภรธนบูรณ์
**ตำแหน่ง:** ครู
**โรงเรียน:** โรงเรียนวัดบ้านรุน
**สำนักงานเขตพื้นที่การศึกษาประถมศึกษา:** บุรีรัมย์ เขต 1

---

## 📌 คำชี้แจงภาพรวมหน่วยการเรียนรู้
แผนการจัดการเรียนรู้นี้ประกอบด้วย **ภาระงานนอกเวลาเรียน 1 ชิ้น** (Gathering — สัมภาษณ์ปราชญ์ชาวบ้าน) และ **แผนการสอนในชั้นเรียน 5 ชั่วโมง** (ชั่วโมงละ 50 นาที) โดยเน้นให้นักเรียนเรียนรู้ **กระบวนการก่อสร้างที่ถูกลำดับขั้นตอน** มากกว่าความสวยงามของผลงาน เพื่อบูรณาการทักษะแนวคิดเชิงคำนวณ (Computational Thinking: CT) และพหุปัญญาด้านตรรกะ-คณิตศาสตร์

### 📝 ภาระงานนอกเวลาเรียน: สัมภาษณ์ปราชญ์ชาวบ้าน (Gathering)
**ขั้น GPAS:** Step 1 — Gathering (สังเกต รวบรวมข้อมูล)

| หัวข้อ | รายละเอียด |
|---|---|
| **มาตรฐาน/ตัวชี้วัด** | ว 4.2 ป.6/3 ใช้แหล่งข้อมูลค้นหาข้อมูลอย่างมีประสิทธิภาพ |
| **จุดประสงค์ K** | นักเรียนระบุขั้นตอนการก่อสร้างบ้านจริงจากคำบอกเล่าของปราชญ์ชาวบ้านได้อย่างน้อย 6 ขั้นตอน |
| **จุดประสงค์ P** | นักเรียนสัมภาษณ์และจดบันทึกข้อมูลลงใบงานได้ครบถ้วน |
| **จุดประสงค์ A** | นักเรียนให้ความเคารพและสนใจฟังผู้ใหญ่ในชุมชน |
| **ทักษะ CT เป้าหมาย** | ✅ Decomposition (แยกย่อยขั้นตอนจริง) |

**คำชี้แจงสำหรับนักเรียน:**

ครูมอบหมายให้นักเรียนแต่ละคู่ไปสัมภาษณ์ **ปราชญ์ชาวบ้าน** ในชุมชนของตนเอง (ช่างปูน ช่างไม้ ผู้ปกครองที่เคยสร้างบ้าน หรือผู้รู้ในหมู่บ้าน) หลังเลิกเรียนหรือในวันหยุด โดยใช้ **ใบงาน 1.0 "สืบค้นภูมิปัญญาชาวบ้าน"** ซึ่งมีคำถามนำดังนี้:

| ข้อ | คำถามสัมภาษณ์ |
|:---:|---|
| 1 | สร้างบ้านจริง 1 หลัง ต้องเริ่มจากอะไรก่อน? |
| 2 | หลังจากเคลียร์พื้นที่แล้ว ขั้นตอนถัดไปคืออะไร? |
| 3 | เสาบ้านต้องวางก่อนกำแพงหรือหลังกำแพง? ทำไม? |
| 4 | หลังคาทำตอนไหน? ทำไมถึงทำเป็นลำดับสุดท้ายๆ? |
| 5 | ถ้าสร้างไม่ถูกลำดับ จะเกิดปัญหาอะไร? |
| 6 | เรียงลำดับขั้นตอนทั้งหมดตั้งแต่ต้นจนจบ (ข้อ 1 ถึง ข้อสุดท้าย) |

| สื่อ/อุปกรณ์ | การวัดและประเมินผล |
|---|---|
| ใบงาน 1.0 "สืบค้นภูมิปัญญาชาวบ้าน", ปากกา, กล้องมือถือ (ถ่ายรูปคู่กับปราชญ์ ถ้าได้รับอนุญาต) | ✅ ตรวจใบงาน 1.0 (เกณฑ์ผ่าน: ระบุขั้นตอน ≥ 6 ขั้นตอน + มีชื่อผู้ให้สัมภาษณ์)<br/>✅ ตรวจความครบถ้วนของคำตอบ 6 ข้อ |

### ⏱️ ชั่วโมงที่ 1: นำเสนอข้อมูลจากปราชญ์ — สำรวจ Minecraft — เทียบขั้นตอนจริง vs เกม

### ⏱️ ชั่วโมงที่ 2: วาดแปลนอิสระ — เขียนอัลกอริทึมลำดับก่อสร้าง
**ขั้น GPAS:** Step 2 — Processing (คิดวิเคราะห์ สรุปความรู้) ตอนที่ 2/2

| หัวข้อ | รายละเอียด |
|---|---|

| **จุดประสงค์ K** | นักเรียนอธิบายความหมายของ Abstraction และ Algorithm ได้ |
| **จุดประสงค์ P** | นักเรียนวาดแบบแปลนบ้านมุมมองบน (Top View) ลงบนกระดาษกราฟได้ (ขนาดอิสระ) และเขียนลำดับขั้นตอนก่อสร้าง ≥ 6 ขั้นตอน |
| **จุดประสงค์ A** | นักเรียนให้ความสำคัญกับการวางแผนก่อนลงมือทำจริง |
| **ทักษะ CT เป้าหมาย** | ✅ Abstraction + Algorithm Design |

| ขั้นตอน | เวลา | กิจกรรมครู | กิจกรรมนักเรียน |
|---|:---:|---|---|

| **ขั้นวาดแปลน** | 20 นาที | - แจกกระดาษกราฟ + **ใบงาน 1.1 "Blueprint Designer"**<br/>- สาธิตการวาด Top View สั้นๆ: ใช้สัญลักษณ์ [W] = กำแพง, [P] = เสา, [D] = ประตู, [G] = กระจก<br/>- ย้ำ: ขนาดอิสระ ออกแบบเอง!<br/>- เดินดูรอบห้อง ถามทีละคู่: *"จะวางเสาตรงไหน? ประตูอยู่ด้านไหน?"* | - จับคู่ ระดมสมอง ตกลงขนาดและรูปทรง<br/>- วาดแปลน Top View ลงกระดาษกราฟ<br/>- ระบุตำแหน่งเสา ประตู หน้าต่าง<br/>- เขียนรายการวัสดุหลัก |

| **ขั้นสรุป** | 5 นาที | - สุ่ม 2 คู่มานำเสนอแปลน + อัลกอริทึม<br/>- ถาม: *"ถ้าสลับขั้นตอนที่ 4 กับ 7 จะเกิดอะไรขึ้น?"* (ตั้งเสาหลังมุงหลังคา = หลังคาลอยฟ้า!)<br/>- สรุป: ลำดับสำคัญมาก = Algorithm | - นำเสนอ<br/>- ร่วมวิจารณ์ |

| สื่อ/อุปกรณ์ | การวัดและประเมินผล |
|---|---|
