Step 1: Gathering — สังเกต รวบรวมข้อมูล

ชั่วโมงที่ 1-2
▼
✅ DECOMPOSITION ✅ PATTERN RECOGNITION
  • ดูวิดีโอ "Smart City in Minecraft" เป็นแรงบันดาลใจ
  • เรียนรู้พื้นฐาน Minecraft: เคลื่อนที่ วางบล็อก Creative Mode
  • สำรวจ Minecraft World ที่ครูเตรียมไว้ (มีตัวอย่างเมืองสำเร็จ)
  • Decompose: แยกย่อยเมืองเป็นส่วนประกอบ (ถนน บ้าน สวน ระบบไฟ)
  • Pattern: จดจำรูปแบบการวางบล็อกที่ซ้ำ สมมาตร ลวดลาย
  • ค้นหาข้อมูล Smart Village จากอินเทอร์เน็ต (ว 4.2 ป.6/3)
★ Logical-Mathematical Spatial Interpersonal
🧠

Step 2: Processing — คิดวิเคราะห์ สรุปความรู้

ชั่วโมงที่ 3-4
▼
✅ ABSTRACTION ✅ ALGORITHM DESIGN
  • Abstract: วาด Blueprint แบบแปลนเมืองบนกระดาษกราฟ
  • Algorithm: เขียนขั้นตอนการสร้าง Step-by-Step (Pseudocode)
  • คำนวณจำนวนบล็อกที่ต้องใช้ (พหุปัญญาด้านตรรกะ-คณิตศาสตร์)
  • เรียนรู้ Redstone เบื้องต้น: Lever, Button, Dust, Repeater, Piston
  • ฝึกสร้างวงจร Redstone ง่าย ๆ (ประตูอัตโนมัติ)
  • เชื่อมโยง Redstone กับ Logic Gate: AND, OR, NOT
  • แบ่งบทบาท: สถาปนิก, วิศวกร Redstone, ผู้จัดการทรัพยากร, ผู้บันทึก
★ Logical-Mathematical Spatial Intrapersonal

Step 3: Applying 1 — ลงมือปฏิบัติ สรุปความรู้

ชั่วโมงที่ 5-6
▼
✅ ALL 4 CT COMPONENTS
  • เปิด LAN Server — ทุกกลุ่มสร้างเมืองพร้อมกัน Multiplayer
  • สร้างเมืองอัจฉริยะตามแบบแปลน: อาคาร 3+ หลัง, ถนน, สวน
  • สร้างระบบ Redstone อย่างน้อย 1 ระบบ (ประตูอัตโนมัติ/ระบบไฟ/สัญญาณเตือน)
  • Debug: แก้ปัญหาที่เกิดขึ้น บันทึกปัญหาและวิธีแก้ไข
  • Mini Design Review ชั่วโมงละ 1 ครั้ง

⭐ SECTION 07 — ปัจจัยความสำเร็จ
อะไรทำให้นวัตกรรมนี้สำเร็จ?
🎮

Game = Motivation

Minecraft สร้างแรงจูงใจภายในอย่างทรงพลัง นักเรียนที่เคยไม่สนใจ กลับกระตือรือร้น

🔴

Redstone = Coding Bridge

ระบบ Redstone เปรียบเสมือน "โปรแกรมมิ่งแบบ Physical" — Bridge สู่ Coding จริง

📐

GPAS = Structure

กรอบ GPAS 5 Steps ทำให้ "การเล่นเกม" กลายเป็น "การเรียนรู้อย่างมีระบบ"

👫

Peer Teaching

นักเรียนที่เก่ง Minecraft เป็น "ครูน้อย" ช่วยสอนเพื่อน — พัฒนาทั้งผู้สอนและผู้เรียน

🧠

CT ซ่อนอยู่ทุกขั้นตอน

นักเรียนใช้ Decomposition, Pattern, Abstraction, Algorithm โดยไม่รู้ตัว!

🏫

ผู้บริหารสนับสนุน

ห้อง Lab พร้อมเครื่อง, LAN Server มั่นคง, PLC แลกเปลี่ยนเรียนรู้

📚 บทเรียนที่ได้รับ (Lessons Learned)
#1 นักเรียนไม่มีประสบการณ์ Minecraft → จัด Tutorial + Buddy System ▼
จัดกิจกรรม Tutorial ก่อนเริ่มหน่วยการเรียนรู้ 1 ชั่วโมง และจับคู่ Buddy ให้นักเรียนที่เก่ง Minecraft ช่วยสอนเพื่อน ผลคือนักเรียนทุกคนสามารถใช้งาน Minecraft ได้ภายใน 1 ชั่วโมง
#2 เพลิดเพลินจนลืมเป้าหมาย → Timer + Design Review + Rubric ▼
ใช้ Timer แจ้งเตือนทุก 15 นาที จัด Mini Design Review ให้กลุ่มรายงานความคืบหน้า และแจก Rubric ตั้งแต่ต้น ทำให้นักเรียนรู้ว่าจะถูกประเมินอะไร จึงมุ่งเน้นไปที่เป้าหมาย
#3 คอมพิวเตอร์ช้า → ปรับ Graphics + Texture Pack เบา ▼
ปรับ Render Distance เหลือ 8 chunks, ปิด Fancy Graphics, ใช้ OptiFine Mod ช่วยเพิ่ม FPS ผลคือเครื่องที่ช้าที่สุดยังทำงานได้ราบรื่น
#4 LAN ขัดข้อง → Backup World + Plan B ▼
เตรียม Backup World ทุกชั่วโมง และมี Plan B ให้ทำงาน Single Player ได้ในกรณี LAN ขัดข้อง เตรียม Hotspot จากมือถือไว้สำรอง
